三分钟搞定补码
𝓒𝓸𝖒𝗽𝖚𝔱𝖊𝓻 𝓢𝓬𝒾𝖊𝖓𝓬𝖊

先来看看原码、反码、补码定义是什么

  • 此处不想看或沒時間看的可以跳过直接看结论

    原码

    每一个整数存入计算机时均是存储其二进制形式，
    但是区分正负数必须要另有一位表示符号，
    于是 [x]原 = 符号位 + 绝对值的二进制，
    如下示例：
    [+1]原 = 0 0000001
    [-1]原 = 1 0000001
    注：此篇均为 8位二进制数 的表示

    反码

    如若为正 [x]反 = 符号位 + 绝对值的二进制，
    如若为负 [x]反 = 符号位为1 + 绝对值的二进制各位取反，
    [+1]反 = 0 0000001
    [-1]反 = 1 1111110
    则为反码，反码的好处此处不讲（反码不常见，不如补码好用）

    补码

    如若为正 [x]补 = 符号位 + 绝对值的二进制
    如若为负 [x]补 = 符号位为1 + 绝对值的二进制各位取反+1，
    [+1] = [00000001]原 = [00000001]反 = [00000001]补
    [-1] = [10000001]原 = [11111110]反 = [11111111]补

现在解释为什么补码有优势

  • 此处不想看或沒時間看的可以跳过直接看结论

    引入概念

-1-概念引入：数据 的 数据原值
对于计算机内的一个 n位（n个0/1）二进制存储空间上的 数据 ，
如果把它看成一个n位正的二进制整数，其数值称为存储数据的数据原值。
如00000001的数据原值为1；10000001的数据原值为129即2的7次方加1。

-2-概念引入：数据原值 在 某码 下的 表示数
按照某码的储存方式储存的 数据 的 数据原值 原本的想表示的值。
如数据原值+1（0 0000001）在原码的存储方式下的表示数为+1；数据原
值+257（1 0000001）在原码的存储方式下的表示数为-1。

补码组成的巧妙之处

分析：
1）补码的2的n次方个数据原值是与2的n次方个数值相对应的，是
一一映射的，而反码则只对应2的n次方减一个数，原因就是10000000
在反码的储存方式下的表示数为-0.
2）补码的优势此处用加法、乘法来解释：
补码储存方式下的 数据原值 与 表示数 对应关系如下：

数据原值 表示数 符号位 当数据原值增加1，表示数的变化
0 ~ 2^(n-1)-1 0 ~ 2^(n-1)-1 0 增加1
2^(n-1) ~ 2^n-1 -2^(n-1) ~ -1 1 增加1

此表请自行证明，是补码精华所在。
另外，对于原码：

数据原值 表示数 符号位 当数据原值增加1，表示数的变化
0 ~ 2^(n-1)-1 0 ~ 2^(n-1)-1 0 增加1
2^(n-1) ~ 2^n-1 -1 ~ -2^(n-1) 1 减少1

由此两表对比可以发现，补码的表示两行都是数据原值+1，
表示数就+1的，是 顺序 的对应着；而原码的第一行是数
据原值+1表示数+1，第二行却数据原值+1表示数-1，不是
顺序的对应着–这就导致：
采用补码的储存方式时
（数据原值 mod 2^n ）=表示数
总是成立。

由于 （a mod 2^n）加或乘 （b mod 2^n） =
（a 加或乘 b）mod 2^n ;
故 只需将数据按照其数据原值的相应运算运算后其表示数也相应地运算了。

结论

总之，补码用2^n-x表示-x使得
（ 数据原值 mod 2^n ）= 表示数
对任意值成立，成功地利用了mod的加减乘除性质。

同樣可能須臾而逝
